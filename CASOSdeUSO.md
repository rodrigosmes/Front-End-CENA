### Proj-FrontEnd-Cena

### 1.1 Caso de Uso: S_HEADER

## Ator Principal: Visitante
# HDR_01
# Pré-condições:
- O visitante acessa a página inicial do site do CENA.
- O visitante deseja navegar por uma seção específica do site.
# Fluxo Principal:
- O visitante clica em um dos itens do menu de navegação localizado no cabeçalho (ex.: "Início", "Blog", "Eventos", "Contato").
- O sistema redireciona o visitante para a seção correspondente da página ou para outra página específica.
# Pós-condições:
- O visitante acessa com sucesso a seção desejada, podendo visualizar seu conteúdo e interagir conforme necessário.
# HDR_02
# Pré-condições:
- O visitante acessa a página inicial do site.
- O visitante deseja alterar o idioma do site.
# Fluxo Principal:
- O visitante localiza e clica no botão de seleção de idioma no cabeçalho.
- O sistema exibe uma lista suspensa com as opções de idiomas disponíveis (ex.: Português, Inglês, Espanhol).
- O visitante seleciona o idioma desejado.
# Pós-condições:
- O conteúdo do site é exibido no idioma selecionado, facilitando o acesso para usuários multilíngues.


### 1.2 Caso de Uso: S_HERO

## Ator Principal: Visitante
# HR_01
# Pré-condições:
- O visitante acessou a página inicial do site e chegou à seção principal de destaque.
# Fluxo Principal:
- O visitante visualiza informações introdutórias sobre o CENA, como missão e objetivos.
- O visitante interage com elementos dinâmicos, como botões de "Saiba Mais" ou imagens clicáveis que levam a outras seções do site.
# Pós-condições:
- O visitante obtém uma visão geral do propósito do CENA e pode decidir explorar outras áreas do site.

### 1.3 Caso de Uso: S_ABOUT_CENA

## Ator Principal: Visitante
# ABT_CENA_01
# Pré-condições:
- O visitante acessa a página inicial do site.
- O visitante seleciona a seção "Quem Somos" no menu de navegação.
Fluxo Principal:
- O sistema redireciona o visitante para a seção "Quem Somos", apresentando informações sobre o CENA (missão, visão, valores e história).
- O visitante lê o conteúdo apresentado e, se disponível, interage com imagens ou elementos multimídia relacionados.
# Pós-condições:
- O visitante obtém informações completas sobre o propósito e os valores do CENA.

### 1.4 Caso de Uso: S_EVENTS

## Ator Principal: Visitante
# EVT_01
# Pré-condições:
- O visitante acessa a página inicial do site.
- O visitante deseja visualizar a agenda de eventos do CENA.
Fluxo Principal:
- O visitante clica no link "Eventos" no menu de navegação.
- O sistema exibe a agenda interativa com datas e descrições de eventos, workshops e debates.
- O visitante pode clicar em um evento específico para visualizar mais detalhes e, se aplicável, inscrever-se no evento.
# Pós-condições:
- O visitante acessa informações detalhadas sobre eventos e, se necessário, conclui a inscrição.

### 1.5 Caso de Uso: S_BLOG

## Ator Principal: Visitante
# BLG_01
# Pré-condições:
- O visitante acessa a página inicial do site.
- O visitante deseja explorar o blog do CENA.
# Fluxo Principal:
- O visitante seleciona a seção "Blog" no menu de navegação.
- O sistema redireciona o visitante para a página do blog, onde são exibidos artigos, análises e publicações organizados por categorias e palavras-chave.
- O visitante pode clicar em uma publicação específica para ler o conteúdo completo.
# Pós-condições:
- O visitante consome o conteúdo do blog e pode compartilhar ou comentar (se habilitado).

### 1.6 Caso de Uso: S_CONTACT

## Ator Principal: Visitante
# CNTCT_01
# Pré-condições:
- O visitante acessa a seção "Contato" no site.
# Fluxo Principal:
- O visitante preenche os campos do formulário de contato com informações como nome, e-mail e mensagem.
- O visitante clica no botão "Enviar".
- O sistema confirma o envio da mensagem com uma notificação visual ou modal.
# Pós-condições:
- A mensagem do visitante é enviada e armazenada no sistema para que a equipe do CENA possa respondê-la.
